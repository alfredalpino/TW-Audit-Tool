# Torpedo Web Internationalization

Production-grade multilingual infrastructure for [torpedoweb.org](https://www.torpedoweb.org).

## Architecture

```
lib/i18n/
├── config.ts          # Locale registry, URL prefixes, region selector data
├── detection.ts       # Geo / Accept-Language / cookie resolution
├── middleware.ts      # Edge routing: redirects + UI-locale rewrites
├── load-messages.ts   # Lazy-loaded JSON namespaces with cache
├── server.ts          # Server-side locale + message helpers
├── analytics.ts       # GA4 locale_view / locale_switch events
└── messages/          # Per-locale JSON (common, nav, footer)

components/i18n/
├── LocaleProvider.tsx       # Client context + dynamic html lang/dir
├── RegionSelector.tsx       # Apple-style country/language modal
└── LocaleFooterTrigger.tsx  # Footer entry point
```

## Locale model

| Locale | URL prefix | Content strategy |
|--------|------------|------------------|
| `en-US` | `/` | Full US marketing site (default) |
| `en-IN` | `/en-in` | Dedicated India pages + local SEO |
| `es-MX`, `fr-FR`, `fr-MA`, `de-CH`, `fr-CH`, `tr-TR`, `ru-RU`, `ar-AE` | `/es-mx`, `/fr`, etc. | UI overlay: middleware **rewrites** to en-US pages while preserving public URL |

This avoids duplicating every page file while keeping SEO-friendly prefixed URLs and translated chrome (nav, footer, metadata).

## Detection priority

1. **URL prefix** (manual navigation / region selector)
2. **Cookie** `TORPEDO_LOCALE` (+ `TORPEDO_COUNTRY`)
3. **Accept-Language** header
4. **Geo-IP** (Vercel `request.geo` / `x-vercel-ip-country`)

Bots and Lighthouse skip geo redirects to protect Core Web Vitals scores.

### Region rules

| Region | Default locale |
|--------|----------------|
| India | `en-IN` (English only) |
| US, UK | `en-US` |
| Mexico | `es-MX` |
| France | `fr-FR` |
| Morocco | `fr-MA` |
| Switzerland | `fr-CH` (French cantons) or `de-CH` |

| Russia | `ru-RU` |
| UAE / Saudi | `en-US` (Arabic `ar-AE` ready) |

## SEO

- `buildLanguageAlternates()` emits hreflang for all configured locales + `x-default`
- Sitemap includes `alternates.languages` on globally localized routes
- JSON-LD `inLanguage` is locale-aware via `lib/seo/schema.ts`

## Adding a locale

1. Add entry to `locales` in `lib/i18n/config.ts`
2. Create `lib/i18n/messages/<lang>/{common,nav,footer}.json`
3. Map import in `lib/i18n/load-messages.ts`
4. Run `npm run i18n:validate`

## Adding page content translations

UI-only locales use English body copy until dedicated content exists. For full page translation:

- **Option A:** Add mirrored routes under `app/(marketing)/<prefix>/` (like `en-in`)
- **Option B:** Migrate to `app/[locale]/(marketing)/` dynamic segment (future)

## Developer commands

```bash
npm run i18n:validate   # Check message key parity
npm run build           # Verify middleware + rewrites compile
```

## Analytics

- `locale_view` — fired on route change with `locale` + `page_path`
- `locale_switch` — fired when user picks a region in the modal

## Cookies

| Name | Purpose | TTL |
|------|---------|-----|
| `TORPEDO_LOCALE` | User-selected BCP-47 locale | 1 year |
| `TORPEDO_COUNTRY` | Selected country id | 1 year |

## Middleware flow (`proxy.ts`)

```
updateSession (Supabase)
  → maybeRedirectToPreferredLocale (cookie / browser / geo)
  → maybeIndiaGeoRedirect (legacy IN fallback)
  → applyLocaleRouting (rewrite UI locales, set x-torpedo-locale header)
```

## Performance

- Message bundles loaded per locale on demand (cached in memory)
- No global i18n payload in main bundle
- UI locales rewrite to existing static pages (no duplicate RSC trees)
